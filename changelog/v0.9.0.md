# v0.9.0 — Pagination aliases + eval infrastructure

_Released 2026-04-26_

## Highlights

- **Canonical pagination field aliases** on `pbir_list_pages` and
  `pbir_list_visuals`. Every paginated response now ships both the legacy
  fields (`total`, `truncated`, `nextOffset`) and the MCP-canonical names
  (`total_count`, `has_more`, `next_offset`). Output schemas accept both;
  `scripts/test-pagination.js` asserts the pairs stay in lockstep across
  every offset boundary. Legacy fields will be removed in a future major —
  prefer the canonical names in new agent prompts.

- **Headless / eval mode via `PBIR_REPORT_PATH`.** Set the env var to a
  `.Report` folder and the server auto-binds at startup, no `pbir_set_report`
  call required. The CLI arg form (`node dist/index.js <path>`) still works
  and wins when both are set; invalid paths log to stderr and leave the
  server running unbound rather than crashing.

- **Evaluation suite** (`evals/`) — closes the fourth phase of the
  mcp-builder workflow that was previously missing. A frozen 20-file,
  32 KB `sample.Report` fixture (3 pages, 10 visuals, custom 4-color theme,
  2 bookmarks, 2 page filters) plus 10 read-only verification questions in
  `questions.xml`, plus a vendored stdio runner (`evals/run.py` from the
  mcp-builder skill) wired up as `npm run eval`. The fixture builder
  patches `crypto.randomBytes` and `Date.now` for byte-identical rebuilds
  so questions stay stable across regenerations. Eval is opt-in (it costs
  Anthropic API tokens) and is **not** part of `test:all`.

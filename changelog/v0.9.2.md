# v0.9.2 — Small polish (read enrichments + checklist skill + gotchas)

_Released 2026-05-02_

Eight small wins from the v0.9.1 release-gate eval Opus left in `BACKLOG.md` ship together as one non-breaking patch. Item 6 (README screenshots) is still open — it's a manual capture step.

## Highlights

### Read tool enrichments

- **`pbir_list_pages` slim mode now returns `width` + `height` per page.** Previously this required `slim:false` — a paper-cut on the eval (Opus reached for canvas dimensions on the first task and had to upgrade the call). Cheap field, common lookup.
- **`pbir_list_pages` response gains top-level `totalVisualCount`.** Sum of `visualCount` across the FULL page set (not just the visible slice when paginated). Avoids client-side paging-and-summing for cross-report decisions.
- **`pbir_get_report` includes `hasSemanticModel: boolean`.** Derived from `findSemanticModelPath()` (try/catch — throw means false). Replaces the failure mode from eval task 9 where Opus called `pbir_model_usage` just to discover the report had no semantic model. Description on `pbir_model_usage` updated to direct callers to check this first.

### Visual filtering

- **`pbir_list_visuals` accepts optional `visualType: string`.** Case-sensitive; filters BEFORE pagination so `total` reflects the filtered count and pagination math stays correct. Pair with per-page iteration when sweeping the report for a single type.

### Description-only

- **`pbir_list_pages` description steers cross-page queries toward `includeVisuals: true`.** One call instead of N parallel `pbir_list_visuals`. Same data, fewer round-trips. Mirrored in `skills/pages.md` and `skills/visuals.md`.

### Skills + docs

- **New `skills/post-edit-checklist.md` (doc-version 1.0).** 7-step verification ported from pbi-pilot's "Mandatory Post-Edit Visibility Checklist (PBIR)". Adapted: `Validate-PBIP.ps1` → `npm run test:all` + in-session `pbir_list_pages({ includeVisuals: true })` / `pbir_list_filters` re-checks; `Restart-PBIDesktop.ps1` → `pbir_reload_report({ confirm: true })`. Wired into `pbir_guide` topic priority between `pages` and `shapes`.
- **`skills/visuals.md` v2.0 → v2.1 — queryState role lookup table at the top.** 28-row table sourced from `VISUAL_BUCKETS` in `src/pbir.ts`, columns `Visual type | Required / valid buckets | Notes`. Saves a round-trip to `pbir_get_visual_types` when picking the right bucket for a new visual.
- **`docs/pbir-gotchas.md` — new "PBI Desktop Quirks" section.** Q1 `enableAutoRecovery` masking PBIR file changes (workaround: temporarily disable in `.pbip`, restore after verification). Q2 visuals or slicers not appearing after restart (4-step checklist: folders on disk → `pageOrder` registration → full Desktop kill via `pbir_reload_report` → rule out auto-recovery). Q3 `filters: []` empty array on schema 2.1.0+ (use `filterConfig`, not bare `filters`; match the format already in the project).

## Tests

- `scripts/test-pagination.js` — 13 new assertions cover the `visualType` filter against a 200-visual mixed fixture (50 card / 50 slicer / 100 tableEx), plus `width`/`height` in slim entries and `totalVisualCount` at the top level. Pre-existing 42 assertions still green.
- `scripts/test-set-report.js` — 6 new assertions cover `hasSemanticModel: false` on the eval fixture (no `.SemanticModel` sibling) and the new `pbir_list_pages` fields end-to-end through the MCP transport.

`npm run test:all` clean.

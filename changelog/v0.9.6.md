# v0.9.6 — `pbir_validate_wireframe`: layout validator over the wire

Single additive tool. Saves v0.10.0 for the multi-item milestone the
backlog already plans (artifact warnings footer, CANONICAL_LAYOUTS export,
`pbir_verify_project`).

## New tool — `pbir_validate_wireframe`

Wraps `validateWireframe()` from `src/wireframe-validator.ts` (the same
function the unit tests in `scripts/test-wireframe-validator.js` exercise)
and exposes it over MCP.

```jsonc
// validate one page (auto-resolves when only one page exists)
{ "pageId": "00000000000000000001" }

// validate every page in the report
{ "scope": "report" }
```

Returns the typed `WireframeReport` shape verbatim — `{ ok, issues[], stats }`
with codes like `OVERLAP`, `LEFT_MARGIN`, `BOTTOM_MARGIN`, `WRONG_GAP_H`,
`WRONG_GAP_V`, `BANNER_WIDTH`, `COLUMN_MISALIGN`, etc. Read-only,
`openWorldHint:false`. Per-tool `outputSchema` shipped alongside the existing
14 schemas in `src/helpers/outputSchemas.ts`.

Pairs with `pbir_audit_theme_compliance` for full project verification:
themes (compliance) + geometry (validate) on every page.

### Why ship now (not v0.10.0)

The validator is solid (12 test cases pass since v0.9.0), and the
artifact's parked "warnings footer" is the natural consumer for v0.10.0
item 2. Shipping the tool first on its own means agents and the artifact
share one source of truth instead of duplicating ~380 lines of validator
logic client-side.

# v0.9.0 addendum — Wireframe scaffolder artifact

_Added 2026-04-26 (no MCP version bump — `src/` untouched)_

Adds a Cowork artifact, delivered as a recovery-copy HTML source in
`artifacts/README.md`. The user instantiates it in Cowork via
`mcp__cowork__create_artifact`; Claude Code's job ends at producing the
checked-in source.

## What it does

- **Inspect tab** — live structural view of the bound `.Report` via
  `pbir_list_pages({ includeVisuals: true, slim: false })`. Page tabs at
  top, SVG canvas at 0.625× scale (1280×720 → 800×450), one type-coloured
  rect per visual with `<type> · <title>` label, side panel showing the
  `pbir_get_visual` payload on click, manual refresh + auto-poll
  (off / 2s / 5s, default off, paused via Page Visibility API when the
  tab is hidden).

- **Build tab** — 5-card grid of canonical layouts A–E (mini SVG previews).
  Click → modal with page-name input (validated against existing names),
  per-slot visual-type dropdown (sensible defaults baked in per layout),
  auto-reload checkbox. Confirm calls `pbir_create_page`, then batch
  `pbir_add_visual({ visuals: [...] })`, then optionally
  `pbir_reload_report({ confirm: true })`. Per-step success/failure
  surfaced inline; on success, switches to Inspect tab with the new page
  selected.

- **Top-bar Reload button** — calls `pbir_reload_report({ confirm: true })`
  with a tooltip warning about unsaved PBI Desktop work.

## Constraints honoured

- Single HTML file, React + Babel only (artifact host provides them).
- No `localStorage` / `sessionStorage` — all state in-memory.
- CSS variables for colours, transparent background, no top-level padding.
- Layout geometry mirrored from `skills/wireframes.md` (Layouts A–E) /
  `scripts/test-wireframe-validator.js`. Comment in the source file points
  back to the canon — drift either side will need a hand-merge.
- No bindings UI in v1 — visuals scaffold empty as designed.
- MCP version unchanged at 0.9.0; no `src/` files modified.

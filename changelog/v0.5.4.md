# v0.5.4 ‚Äî Calculation Groups Tab & Light Mode

**Date:** 2026-04-14

## Calculation Groups Tab

The usage dashboard now parses and displays calculation groups from both TMDL and BIM models.

- New **Calc Groups** tab between Functions and Pages, with a badge showing the count
- Each group renders as a collapsible page-card showing:
  - Group name and description
  - **Items** count and **Precedence** value
  - Per-item cards with ordinal, name, `///` TMDL description, and the full DAX expression
  - Format string expressions shown as inline `fmt:` badges when present
- TMDL parser handles multi-line calc item expressions (depth-3 continuation lines)
- The implicit `Name` column of a calc group table is correctly excluded from the Columns tab

## Light Mode

The dashboard now supports a light theme alongside the default dark theme.

- **Toggle button** (`‚òæ` / `‚òÄ`) in the header, next to Refresh
- **Persistence** via `localStorage['usage-theme']` ‚Äî survives reloads and auto-refresh cycles
- **No FOUC** ‚Äî an inline `<script>` in `<head>` sets `data-theme` before the body paints
- Theme is driven by CSS custom properties in `:root` and `[data-theme="light"]`:
  - Backgrounds (`--bg`, `--surface`, `--surface-alt`, `--surface-center`)
  - Borders (`--border`, `--border-soft`, `--border-row`)
  - Text (`--text`, `--text-body`, `--text-muted`, `--text-dim`, `--text-faint`, `--text-fainter`)
  - Chip palettes (`--chip-dep-*`, `--chip-used-*`)
  - Syntax highlighting (`--code-name`, `--code-type`, `--code-punct`) for UDF parameter signatures
- All inline JS styles that hardcoded dark-specific colors were moved to classes or switched to `var(--‚Ä¶)` so both themes render correctly

## KPI Tooltips

Hover any summary tile to see a custom tooltip explaining what it means:

- **Direct** ‚Äî "Fields bound to at least one visual‚Ä¶"
- **Indirect** ‚Äî "Not on any visual, but referenced by direct measures via DAX‚Ä¶"
- **Unused** ‚Äî "Not referenced anywhere in the report ‚Äî safe to remove"
- Tooltips explain the `NM ¬∑ NC` shorthand (measures vs columns)
- First and last tiles align their tooltips to the grid edge to avoid viewport clipping
- Uses `.has-tip[data-tooltip]` with `::before` / `::after` pseudo-elements ‚Äî no JS tooltip library

## Copy-DAX Button

Every DAX code block (`.lineage-dax`) gets a small `‚éò` button in the top-right, visible on hover.

- Used on measure lineage expressions, UDF function bodies, and calc group item expressions
- Click copies the raw DAX to the clipboard ‚Äî no selection needed
- Confirms with a green `‚úì` for 1.5s, then resets
- Graceful fallback: tries `navigator.clipboard.writeText` first, falls back to a hidden-textarea + `execCommand('copy')` for older browsers
- `addCopyButtons()` is called on init and after every `openLineage()` call; idempotent via a `data-copy-wired` marker so re-renders are safe

## Dark-Mode Readability Fixes

Bumped the three dimmest text tones in the dark palette so muted labels are actually readable:

| Variable | Before | After | Used for |
|---|---|---|---|
| `--text-dim` | `#64748B` | `#7A8595` | tab labels, table headers, field table names |
| `--text-faint` | `#475569` | `#5E6A7B` | stat details (`10M ¬∑ 12C`), section titles, page stat labels |
| `--text-fainter` | `#334155` | `#4A5566` | timestamp, header separator, arrow columns |

Also replaced all inline `color:#334155` dash placeholders with `var(--text-faint)` so the `‚Äî` marks in tables are visible. Light mode values unchanged.

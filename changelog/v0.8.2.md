# v0.8.2 — outputSchema tightening + README polish

_Released 2026-04-26_

## Highlights

- **Tightened `outputSchema` on 14 read tools** — `pbir_list_pages`,
  `pbir_list_visuals`, `pbir_get_visual` (slim), `pbir_get_report`,
  `pbir_list_filters`, `pbir_list_bookmarks`, `pbir_list_report_themes`,
  `pbir_get_report_theme`, `pbir_lookup_theme_property`,
  `pbir_audit_theme_compliance`, `pbir_diff_report_theme`,
  `pbir_model_usage`, `pbir_guide`, `pbir_load_tools` now ship per-tool zod
  schemas in the `tools/list` response. Mutation tools and the
  `pbir_get_visual` verbose / `pbir_get_visual_types` / `pbir_get_report_settings`
  raw-dump tools keep the loose `{success, error?}` envelope (their shapes are
  too noisy or not even a JSON object). All schemas use `.passthrough()` and
  mark known fields `.optional()` so adding a new server-side field never
  breaks an existing schema. New shared map: `src/helpers/outputSchemas.ts`
  (`READ_TOOL_SCHEMAS`); `doRegister()` looks up the override and falls back
  to `GENERIC_OUTPUT_SCHEMA` for tools not in the map.

- **Migrated `pbir_model_usage` connection guard to `requireProject` + `fail()`**
  for consistency with the W1.4 pattern used by every other handler. The
  inline `{ content: [{ type: "text", ... }], isError: true }` envelope is
  gone. The explicit `reportPath` override is preserved — `requireProject`
  is only invoked when no override was supplied.

- **README polish** — Cowork plugin install section under Quick Start with
  drag-and-drop instructions and a GitHub Releases link, refreshed version
  badges (0.5.2 → 0.8.2) and tool counts (54 → 56), new "What's new in 0.8"
  callout linking to the changelog, screenshot placeholders (`<!-- TODO -->`
  HTML comments) at the hero spot, batch-mode example, and plugin install
  flow. Dropped the "Why the default flipped in v0.6.2" historical section
  (still in the changelog).

The audit-driven refactor stack lands as a single coherent breaking

## BREAKING

- **All tool names prefixed with `pbir_`.** `add_visual` â†’ `pbir_add_visual`,
  `set_report` â†’ `pbir_set_report`, `list_pages` â†’ `pbir_list_pages`,

- **`pbir_add_visual` single-mode removed.** Use the `visuals: [...]`

  gone, replaced with a stable `code` plus structured payload. See

## Changed

- **Migrated every `server.tool()` registration to
  `server.registerTool()`.** The migration happens entirely in the
  wrapper layer in `src/index.ts` â€” every tool module's existing
  `server.tool()` call is routed through `registerTool` under the
  hood, so no per-tool churn was needed. Each tool now ships with:
  - `title` â€” human-readable name for permission UIs.
  - `outputSchema` â€” a shared loose envelope
    (`{ success: boolean, error?: string }` + passthrough). Satisfies
    the MCP spec's `structuredContent` validation contract universally
    without forcing 55 bespoke schemas; tighter per-tool schemas can
    land in follow-up work without touching the wrapper.
  - `annotations` â€” flow through unchanged from W2.
- **`ok()`/`fail()` in `src/helpers/mcpResult.ts` dual-emit
  `structuredContent`** alongside the legacy `content[0].text`
  envelope. Newer MCP clients consume the parsed object directly with
  no `JSON.parse` round-trip; older clients keep parsing the text
  envelope. The `safe()` wrapper in `src/index.ts` back-fills
  `structuredContent` on any handler that returned a content-only
  envelope, so the dual-emit contract holds across the 100+ inline
  returns scattered across the tool modules.

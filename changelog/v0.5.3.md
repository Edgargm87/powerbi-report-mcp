# v0.5.3 — Usage Dashboard Enhancements

**Date:** 2026-04-13

## Three-Tier Field Classification

Fields are now classified into three tiers instead of binary used/unused:

- **Direct** — bound to a visual (data well, filter, or conditional formatting)
- **Indirect** — not on any visual, but referenced by direct measures via DAX or used in relationships
- **Unused** — not referenced anywhere in the report; split into:
  - **Not Referenced Anywhere** — safe to remove
  - **Dead Chain** — referenced by other measures, but the entire chain never reaches a visual

## Conditional Formatting Detection

The parser now scans the `visual.objects` section recursively, detecting measures and columns used in:

- **Image/icon formatting** (`cardImage.imageData`)
- **Reference labels** (`referenceLabel.value`, `referenceLabelTitle`, `referenceLabelDetail`)
- **Conditional colors, data bars, and other formatting rules**

Previously, only `queryState` (data wells) and `filterConfig` (filters) were scanned. This fix correctly identifies 24+ additional direct measures in a typical report.

## UDF Functions Tab

- Parses user-defined functions from `functions.tmdl` (TMDL) and `model.bim` (BIM)
- Shows function name, parameters, expression, and description
- Counts how many measures reference each function
- Clickable measure chips to jump to lineage view
- UDF references shown as teal `f` cards in the lineage upstream column
- `.About` helper functions are filtered out

## Standalone Usage Dashboard Web App (`usage-app.ts`)

- New `npm run usage:app` command launches a web server with a landing page
- **Native folder picker** — Browse button opens the Windows folder dialog
- **Paste path** — type or paste `.Report` path directly
- **Recent reports** — one-click access to previously analysed reports
- Generates the full interactive HTML dashboard on submit

## Dashboard UI Improvements

- **5 combined summary tiles** — Direct (M+C), Indirect (M+C), Unused (M+C), Pages, Visuals
- **Collapsible sections** in the Unused tab (page-card pattern)
- Descriptions visible in collapsed state explaining what each section means and what action to take
- Count labels show "Measures" or "Columns" per section
- Lineage tab moved to end of tab bar
- Status badges (INDIRECT amber, UNUSED red) on measure and column rows

## Files Changed

- `src/model-usage.ts` — three-tier classification, UDF parsing, conditional formatting scanner, dashboard HTML
- `src/usage-app.ts` — new standalone web app with native folder picker
- `package.json` — added `usage:app` script
- `README.md` — updated tool descriptions, architecture, tips

# v0.8.1 — Pagination + per-skill resources

Two non-breaking quality-of-life additions on top of the v0.8.0 surface.

## Added

- **Pagination on `pbir_list_visuals` and `pbir_list_pages`.** Both tools
  accept optional `limit` (1–500, default 100) and `offset` (default 0)
  and return `total`, `truncated`, and `nextOffset` alongside the sliced
  payload. The dedup-cache key now includes pagination params so paged
  calls don't collide. Calling either tool without `limit`/`offset`
  preserves the prior shape (just with the new pagination fields
  appended) — no breaking changes for existing callers. On
  `pbir_list_pages`, the `pageId` shortcut still bypasses pagination
  (`total: 1, truncated: false, nextOffset: null`).

- **Per-skill MCP resources at `resource://pbir-skill/{topic}`.** One
  resource per non-underscore file in `skills/` — 16 in total, covering
  `calculations`, `elicitation`, `errors`, `filters`, `formatting`,
  `pages`, `report`, `report-design`, `shapes`, `slicers`, `svg-visuals`,
  `themes`, `themes-per-visual`, `token-usage`, `visuals`, `wireframes`.
  Read live from disk on each request, so prose edits don't require a
  server restart. Lets resource-aware clients (Claude Desktop's
  `@`-picker, Cowork) surface individual skills natively without going
  through `pbir_guide(topic)`. The existing
  `resource://pbir-instructions` and `pbir_guide` paths stay; they all
  work in parallel. The `_overview.md` file (underscore-prefixed)
  remains the meta resource and is excluded from the per-skill set.

## Tests

- New `scripts/test-pagination.js` (24 assertions) — synthetic in-memory
  page with 200 fake visuals + 25 fake pages, captures the registered
  handler via a `server.tool` stub, and exercises the offset/limit
  boundary cases plus the `pageId`-shortcut bypass on `list_pages`.
  Wired into `npm run test:all`.
- `scripts/test-tool-surface.js` extended to also issue `resources/list`
  and assert the per-skill resource set matches the non-underscore files
  in `skills/` exactly.

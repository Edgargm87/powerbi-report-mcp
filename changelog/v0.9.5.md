# v0.9.5 — Tier C.1 catalog reduction + pbir_guide fix

Pure existing-API + docs work driven by the v0.9.4 session-cost baseline (catalog
was 14,509 tokens / 54.8% of session). No new tools, no artifact changes,
backward-compatible across the board.

## Fix 1 — `pbir_guide` outputSchema rejection

`pbir_guide` returns markdown text content via `content[0].text` only (no
`structuredContent`), but `READ_TOOL_SCHEMAS` in `src/helpers/outputSchemas.ts`
declared a `guideSchema` for it. The MCP SDK rejected every successful call
with "Output validation error" because the response payload didn't match the
declared structured shape — the tool path was effectively broken in strict
clients.

Fix: drop `pbir_guide` from `READ_TOOL_SCHEMAS`. Text-content tools opt out
of structured-output validation (same pattern as `pbir_get_visual` verbose,
`pbir_get_visual_types`, `pbir_get_report_settings`).

Regression test extended in `scripts/test-set-report.js` — `pbir_guide({topic:
'wireframes'})` must return a non-error response and `content[0].text` must
start with a markdown header.

## Perf 1 — `pbir_get_visual_types` slim default

The handler returned the full `VISUAL_BUCKETS` map verbatim — ~1,166 tokens
for what is logically a static enum list. The bucket metadata is only useful
when picking a visualType you're about to bind, and `pbir_lookup_theme_property`
already covers per-type schema lookups for formatting.

Default now returns `{success, types:[...sorted], count}` (~400 tokens incl.
JSON-RPC envelope). Pass `verbose:true` to get the full `{type: [buckets]}`
map (back-compat for any caller relying on the fat shape). Tool description
documents the dual shape.

## Perf 2 — Top-3 catalog hog descriptions trimmed

Trimmed `registerTool` descriptions and `inputSchema.describe()` strings on
the three biggest catalog consumers from the v0.9.4 baseline:

| Tool | v0.9.4 | v0.9.5 | Saved |
|------|-------:|-------:|------:|
| pbir_add_visual    | 1,405 | 1,381 | -24 |
| pbir_layout_grid   |   747 |   621 | -126 |
| pbir_list_pages    |   646 |   506 | -140 |

Kept the gotchas (Series binding requirement, KPI=card+measure, scatter
bucket, totalVisualCount sums full set, canvas geometry); dropped redundant
phrasing and self-evident param docs.

Bug fix tucked into the trim: `pbir_add_visual`'s description previously
claimed scatter uses the `Details` bucket — that's wrong. `VISUAL_BUCKETS`
in `src/pbir.ts` and the data-goblin reference both have scatter on
`Category`; `Details` was dropped silently by Desktop. Description now
reads `Category`.

`pbir_add_visual`'s 1.4k cost is dominated by the inline `VisualSpecSchema`
(one `z.array(z.object({...37 fields...}))`) which ships verbatim in the
catalog regardless of description length — description-only trim hits
diminishing returns there. Deeper savings on this tool would require
splitting `VisualSpecSchema` and is left for a future pass.

## Docs — `MCP_TOOLS=minimal` opt-in surfaced earlier

The README "Smart Tool Loading" section already documented `MCP_TOOLS=minimal`,
but it lived deep in the doc. Added a copy-pasteable jsonc snippet right
after the install/configure section with a short trade-off summary, including
the gotcha that `notifications/tools/list_changed` isn't honoured by
Claude Desktop/Code so activated tools don't surface mid-session in those
clients.

## Measured impact

From `measurement/session-baseline.md` (re-run on the v0.9.5 build):

| Bucket | v0.9.4 | v0.9.5 | Delta |
|--------|-------:|-------:|------:|
| Tool catalog (tools/list)        | 14,509 | 14,216 | -293 |
| Tool result payloads (11 calls)  |  5,702 |  4,939 | -763 |
| Skill content loaded             |  5,936 |  5,936 |  0 |
| **Total per representative session** | **26,488** | **25,432** | **-1,056** |
| Catalog as % of session          |  54.8% |  55.9% | +1.1pp |

The biggest single win was actually on the call-payload side: `pbir_get_visual_types`
(`{}`) dropped from 1,166 → 403 tokens, accounting for ~760 of the ~1,000 tokens
saved overall. The catalog itself only dropped ~290 tokens because
`pbir_add_visual`'s schema (not its description) is the elephant.

The catalog-as-%-of-session number went UP slightly because the total session
shrank faster than the catalog did — consistent with the call-payload win
being larger than the catalog trim.

New top-3 catalog hogs after the trim:

| Rank | Tool | Catalog tokens | % of catalog |
|------|------|---------------:|-------------:|
| 1 | pbir_add_visual    | 1,381 | 9.7% |
| 2 | pbir_layout_grid   |   621 | 4.4% |
| 3 | pbir_bulk_bind     |   534 | 3.8% |

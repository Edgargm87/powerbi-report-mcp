# Changelog — v0.5.1 (2026-04-11)

## [0.5.1] — 2026-04-11

### Added
- **`model_usage`** tool — cross-references the semantic model (TMDL or BIM) with report visuals to show where every measure and column is used, DAX dependencies, reverse dependencies, unused fields, and per-page breakdowns
- **HTML dashboard** — self-contained dark-themed dashboard with 5 tabs (Measures, Columns, Pages, Lineage, Unused), generated to `.usage/<report-name>/index.html`
- **Standalone CLI** (`usage-cli.ts`) — one-shot mode generates dashboard and opens in browser; watch mode (`--watch`) adds HTTP server + `fs.watch` with auto-regeneration on file changes
- **Cache invalidation** — 9 visual-modifying tools (`add_visual`, `delete_visual`, `duplicate_visual`, `change_visual_type`, `update_visual_bindings`, `bulk_bind`, `bulk_delete_visuals`, `delete_page`, `duplicate_page`) call `invalidateCache()` to trigger background regeneration
- **File watchers** — `fs.watch` on both `.Report` and `.SemanticModel` folders with 500ms debounce, started automatically on `set_report`
- **Dual model format support** — parses both TMDL (text-based `.tmdl` files) and BIM (`model.bim` JSON)
- **DAX dependency parsing** — regex-based `[MeasureName]` detection against known measure names
- **HierarchyLevel binding support** — scans hierarchy-based fields in both queryState projections and filterConfig
- **Filter scanning** — extracts Measure, Column, and HierarchyLevel field references from visual `filterConfig.filters`
- npm scripts: `usage` (one-shot), `usage:watch` (watch mode)
- `.usage/` added to `.gitignore`
- `model_usage` added to `DEFAULT_TOOLS` (always available, no need to call `load_tools`)

### Fixed
- **Calculated column name parsing** — TMDL calculated columns like `column Channel = IF(...)` were stored with the full DAX expression as the name; now correctly strips `= ...` suffix
- **HierarchyLevel not scanned** — visuals using hierarchy-based slicers (e.g. `Product[Category]` via a hierarchy) were not detected, causing wrong `isSlicerField` and undercounted usage
- **Measures in filterConfig ignored** — filter scanning only checked for Column fields; now also scans Measure and HierarchyLevel fields (37 missed references fixed in training report)

### Changed
- `scanReportBindings()` refactored to use `PbirProject` class instead of duplicating directory walking and JSON parsing
- Field extraction logic consolidated into shared `extractFieldRef()` helper used by both queryState and filterConfig scanning
- Total tools: **48** (+ `load_tools`) — 11 default, 37 on-demand
- Version bumped to 0.5.1

### Architecture
- New files: `src/model-usage.ts` (~1000 lines), `src/usage-cli.ts` (~120 lines)
- Modified: `src/index.ts` (imports, DEFAULT_TOOLS, registration), `src/tools/visuals.ts`, `src/tools/bindings.ts`, `src/tools/bulk.ts`, `src/tools/report.ts` (invalidateCache calls)
- Output directory: `.usage/<report-name>/` inside the MCP project root (not in the report folder)

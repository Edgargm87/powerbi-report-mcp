# v0.5.8 — Canonical L/R Margin 20px → 15px

**Date:** 2026-04-15

A small but cross-cutting fix that recovers **10 extra pixels of usable width** on every page (1240 → 1250) and resolves a long-standing inconsistency between `skills/wireframes.md` and `skills/report-design.md`.

## The Problem

`skills/wireframes.md` v2.0 declared the L/R margin as 20px (1240 usable). `skills/report-design.md` v2.0 declared it as "20px left/right, 20px top/bottom" — but the wireframe validator (`src/wireframe-validator.ts`) had `marginTop: 0, marginBottom: 6`, so the *top/bottom* part of report-design.md was wrong, and the two files disagreed about where the safe area starts.

While in there, we asked the more interesting question: **why 20?** The validator allows 6px on the bottom for breathing room, the gap between visuals is 5px, and visuals lose nothing visually at 15px from the page edge. Tightening to 15 buys a clean +10 usable px on every page — enough to make the 5-card row honest at 246 (was 244, with stranded right-edge pixels) and the 3-column row honest at 413/413/414 (was 410/410/410).

## The Change

L/R margin is now **15px** everywhere. Top margin stays 0, bottom margin stays 6, gap stays 5, banner stays exempt.

| Property            | Before | After |
|---------------------|--------|-------|
| Margin left         | 20     | **15** |
| Margin right        | 20     | **15** |
| Margin top          | 0      | 0     |
| Margin bottom       | 6      | 6     |
| Usable width        | 1240   | **1250** |
| Usable height       | 714    | 714   |
| Gap                 | 5      | 5     |
| Banner              | (0,0,1280,52) | (0,0,1280,52) |
| First content row y | 57     | 57    |

## What Changes for Users

- **Existing reports built before v0.5.8 still validate**, because the validator only enforces `x ≥ marginLeft` and `right edge ≤ width − marginRight`. A visual at x=20 with right edge 1260 is still valid against the new 15px margin (just slightly off-edge).
- **New reports** built via the canonical layouts will land 5px further out on each side, giving 10px of extra usable width per page.
- **`guide("wireframes")`** now returns the recomputed v2.1 layouts.
- **`guide("report-design")`** now returns principles only and refers mechanics to wireframes.

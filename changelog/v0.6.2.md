# v0.6.2 — Default tool loading flipped

## What changed

The MCP server now loads **all 55 tools at startup by default**. The previous tiered mode (12 default + 42 on-demand via `load_tools`) is still available as an opt-in via `MCP_TOOLS=minimal`.

## Why

The tiered design assumed MCP clients would refresh their tool catalog when the server sent `notifications/tools/list_changed`. In practice, most clients — Claude Desktop especially — snapshot the catalog at session start and never re-fetch. That meant `load_tools` could activate a tool server-side (e.g. `delete_page`) but the client would never see it, leaving the user unable to invoke the activated tool.

Loading all tools upfront adds ~7,500 tokens of schemas but guarantees every tool is invokable regardless of client behavior. In a 200k context window that's ~3.75% — negligible for most workflows, and easily dwarfed by a single `model_usage` response.

## Opt-in: `MCP_TOOLS=minimal`

Long Claude Code sessions on a tight context budget can still use the tiered mode:

```json
{ "env": { "MCP_TOOLS": "minimal" } }
```

Loads the 12 core tools (`set_report`, `list_pages`, `list_visuals`, `create_page`, `add_visual`, `get_visual`, `format_visual`, `update_visual_bindings`, `set_report_theme`, `bulk_bind`, `model_usage`, `reload_report`) plus the `load_tools` meta-tool. Activate the rest on demand.

## Other changes

- **`load_tools` response now includes a `refreshHint`** — clarifies that `tools/list_changed` was sent and suggests falling back to the default mode if tools don't appear.
- **stderr diagnostic logging** — `[load_tools] Activated: <name>` per tool, plus a final count. Useful for debugging client-side refresh issues.
- **Legacy compatibility** — `MCP_TOOLS=all` still works (it's now equivalent to the default).

## Migration

- **Claude Desktop users**: drop the `"env": { "MCP_TOOLS": "all" }` block from your config — it's the default now. No other changes needed.
- **Claude Code users**: no change required. If you want the old tiered behavior, add `"env": { "MCP_TOOLS": "minimal" }` to your config.

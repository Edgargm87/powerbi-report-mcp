# Changelog — powerbi-report-mcp

Each release has its own file in [`changelog/`](changelog/).

| Version | Date | Highlights |
|---------|------|------------|
| [**0.5.7**](changelog/v0.5.7.md) | 2026-04-15 | **Skill content rewrite to 100% coverage** — eight stale skill files (`slicers`, `visuals`, `formatting`, `filters`, `pages`, `themes`, `report`, `token-usage`) rewritten v1.0 → v2.0 against the actual TypeScript source. Drives the v0.5.6 audit baseline from **17 missing tools (69.1%) → 0 missing (100.0%)**. New surface coverage: `change_visual_type`/`delete_visual`/`get_visual_types`/`bulk_delete_visuals`, `set_conditional_format`/`set_visual_sort`, `format_visual` auto-routing with the explicit `CONTAINER_CATEGORIES` set, `set_datapoint_colors` category-mode (`categoryEntity`/`categoryProperty`), advanced filter type with And/Or compound operators, `set_page_background`/`set_filter_pane`/`set_visual_interaction`/`get_page_summary`/`manage_extension_measures`, `diff_report_theme`/`list_report_themes`/`audit_theme_compliance`, `model_usage`/`load_tools`/12-default-42-on-demand tool loading model, refreshed token-usage numbers. Sets up v0.5.8 to wire `audit-skill-coverage.js --strict` into pre-commit/CI. |
| [**0.5.6**](changelog/v0.5.6.md) | 2026-04-15 | **Skills plumbing & coverage audit** — `guide` tool now reads `skills/*.md` from disk at runtime instead of from embedded TS string constants (svg-visuals + report-design extracted to MD, guide.ts shrank 337 → 95 lines). New `scripts/audit-skill-coverage.js` cross-references every registered tool against backtick mentions in `skills/*.md`, prints a coverage table, supports `--json` and `--strict` (exit 1 on any gap) for CI. Baseline: **55 tools, 38 covered (69.1%), 17 missing** — the work list for v0.5.7's stale-skill rewrites. Topic discovery is dynamic — adding a new `skills/*.md` requires no code change. |
| [**0.5.5**](changelog/v0.5.5.md) | 2026-04-15 | Slicer **multi-select** support — new `multiSelect` param on `add_visual` writes `objects.selection.singleSelect`, and `get_visual` slim mode now surfaces `slicerMode` + `multiSelect` for all slicer types (with PBI-default fallback). Applies to `slicer` and `listSlicer`. **`reload_report` moved to `DEFAULT_TOOLS`** — the on-demand `load_tools` mechanism activates server-side but most LLM harnesses snapshot the MCP catalog at session start, so a lazy `reload_report` could be activated but never invoked. Defaulting closes the trap. |
| [**0.5.4**](changelog/v0.5.4.md) | 2026-04-14 | Usage dashboard: **Calc Groups** tab (TMDL + BIM parser, per-item DAX expressions, precedence), **Tables tab** (PK/FK/relationship rendering, inferred PK detection), **light mode** with toggle and localStorage persistence, **KPI tooltips**, **copy-DAX button** on code blocks, **HIDDEN badge** for tooltip/drillthrough pages, dark-mode readability bump. **Wireframe validator** (`src/wireframe-validator.ts`) — pure function that catches out-of-bounds, overlaps, wrong margins/gaps, silent (0,0) defaults, rounding overflow, and the new **6px bottom margin** rule. **Shape text** — `createVisual.ts` now emits labels into the `objects.text` branch (was silently ignored via `general.paragraphs`), with friendly font names, bold/italic/underline, alignment, padding. **Skill rewrites** (`skills/shapes.md` v2.0 + `skills/wireframes.md` v2.0) matching the canonical constants, and a Python builder for the 5-page wireframe test report |

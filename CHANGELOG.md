# Changelog — powerbi-report-mcp

Each release has its own file in [`changelog/`](changelog/).

| Version | Date | Highlights |
|---------|------|------------|
| [**0.6.0**](changelog/v0.6.0.md) | 2026-04-15 | **Skills & safety polish.** No new tools — every change is something an agent will *do differently* next session. **Bulk-ops safety gate**: `bulk_delete_visuals` / `bulk_update_format` / `bulk_bind` now return a structured error when the operation would touch more than 5 visuals unless `confirmBulk: true` is set — guards against the "pipe every id from list_visuals into bulk_delete" page-wipe failure mode. **Wireframe cross-row column alignment**: `src/wireframe-validator.ts` gains a constrained `COLUMN_MISALIGN` warning that fires when two adjacent rows of the same visual count drift horizontally beyond a 1px tolerance (warning-level, ignores different-count or non-adjacent rows, silent on rounding noise). Test suite 12→13 with new `BAD 8` case and `runExpectCode()` variant. **New skill: `skills/calculations.md`** — documents the parked visual-calculations state and points DAX authoring at `powerbi-modeling-mcp` / daxpatterns.com. **New skill: `skills/themes-per-visual.md`** — 250-line per-visual-type theme JSON reference covering the `visualStyles` block (property shapes, category catalog by visual family, gotchas, worked examples), sourced from Microsoft's public theme schema. **`skills/svg-visuals.md`** gains a "Where the DAX Actually Lives" section making the modeling-MCP boundary explicit. **`skills/report-design.md` v2.1 → v2.2** — new 5-step mental-model prelude ("name the page's one job → pick the headline → context before detail → delete rather than add → pre-attentive attributes are a budget"). Version bump 0.5.9 → 0.6.0 in `package.json`, `package-lock.json`, `src/index.ts`. `npm run test:all` clean, audit still 55/55 100.0%. |
| [**0.5.9**](changelog/v0.5.9.md) | 2026-04-15 | **Audit CI gate + pre-commit hook** — the skill-coverage audit and wireframe validator are now run automatically on every commit (via `.githooks/pre-commit`) and every push/PR (via `.github/workflows/ci.yml`). Undocumented tools or broken layouts can no longer ship. **`DEFAULT_TOOLS` extracted** to `src/default-tools.ts` as a single source of truth — both `src/index.ts` (runtime) and `scripts/audit-skill-coverage.js` read from one file; no more two-location drift. **Audit now reports tiered coverage**: default vs on-demand tools are split in the table and JSON output (`Default 12/12, On-demand 43/43`), tools are tagged `[DEFAULT]` in the listings, and missing-default-tool regressions get an extra warning. New npm scripts: `audit`, `audit:strict`, `test:wireframe`, `test:all`, `hooks:install`. One-shot installer `npm run hooks:install` sets `core.hooksPath` to `.githooks/` so the pre-commit hook is shared across every clone. **CONTRIBUTING.md v1.2 → v1.3** — new "Automated gates" subsection documents the three checks, Step 5 (documentation) is now marked MANDATORY enforced by CI, Step 4 points at `src/default-tools.ts`, setup adds `npm run hooks:install`. **Version hygiene**: `McpServer` version literal bumped 0.5.2 → 0.5.9 in `src/index.ts` (it had been stale for 7 minor versions; the MCP handshake was reporting the wrong version to clients). Pre-commit hook dry run clean, `npm run test:all` clean, audit still 100.0% (55/55). |
| [**0.5.8**](changelog/v0.5.8.md) | 2026-04-15 | **Canonical L/R margin 20px → 15px** — recovers 10 extra usable pixels per page (1240 → 1250) and resolves the wireframes vs report-design contradiction. `src/wireframe-validator.ts` constants updated, all 5 canonical layouts (A–E) recomputed against new constants, all 7 negative test cases re-anchored, all 12 validator tests still pass. Coverage on every layout improves ~0.5–1.2pp (Layout A 93.7%→94.3%, Layout D 93.3%→94.5%). `skills/wireframes.md` v2.0 → v2.1 with new spacing formula `(1250 − (N−1)×5)/N` (N=5 cards now 246 wide, N=3 now 413/413/414). `skills/report-design.md` v2.0 → v2.1 fixes the wrong "20px top/bottom" claim, drops two unvalidated ASCII layouts, becomes principles-only with a pointer to `skills/wireframes.md` for mechanics — clean separation between **principles** (typography, color, KPI patterns) and **mechanics** (margins, gaps, validated coordinates). `src/index.ts PBIR_INSTRUCTIONS` and the deprecated `docs/wireframes.md` also updated. Audit `--strict` still 100.0% coverage (55/55). |
| [**0.5.7**](changelog/v0.5.7.md) | 2026-04-15 | **Skill content rewrite to 100% coverage** — eight stale skill files (`slicers`, `visuals`, `formatting`, `filters`, `pages`, `themes`, `report`, `token-usage`) rewritten v1.0 → v2.0 against the actual TypeScript source. Drives the v0.5.6 audit baseline from **17 missing tools (69.1%) → 0 missing (100.0%)**. New surface coverage: `change_visual_type`/`delete_visual`/`get_visual_types`/`bulk_delete_visuals`, `set_conditional_format`/`set_visual_sort`, `format_visual` auto-routing with the explicit `CONTAINER_CATEGORIES` set, `set_datapoint_colors` category-mode (`categoryEntity`/`categoryProperty`), advanced filter type with And/Or compound operators, `set_page_background`/`set_filter_pane`/`set_visual_interaction`/`get_page_summary`/`manage_extension_measures`, `diff_report_theme`/`list_report_themes`/`audit_theme_compliance`, `model_usage`/`load_tools`/12-default-42-on-demand tool loading model, refreshed token-usage numbers. Sets up v0.5.8 to wire `audit-skill-coverage.js --strict` into pre-commit/CI. |
| [**0.5.6**](changelog/v0.5.6.md) | 2026-04-15 | **Skills plumbing & coverage audit** — `guide` tool now reads `skills/*.md` from disk at runtime instead of from embedded TS string constants (svg-visuals + report-design extracted to MD, guide.ts shrank 337 → 95 lines). New `scripts/audit-skill-coverage.js` cross-references every registered tool against backtick mentions in `skills/*.md`, prints a coverage table, supports `--json` and `--strict` (exit 1 on any gap) for CI. Baseline: **55 tools, 38 covered (69.1%), 17 missing** — the work list for v0.5.7's stale-skill rewrites. Topic discovery is dynamic — adding a new `skills/*.md` requires no code change. |
| [**0.5.5**](changelog/v0.5.5.md) | 2026-04-15 | Slicer **multi-select** support — new `multiSelect` param on `add_visual` writes `objects.selection.singleSelect`, and `get_visual` slim mode now surfaces `slicerMode` + `multiSelect` for all slicer types (with PBI-default fallback). Applies to `slicer` and `listSlicer`. **`reload_report` moved to `DEFAULT_TOOLS`** — the on-demand `load_tools` mechanism activates server-side but most LLM harnesses snapshot the MCP catalog at session start, so a lazy `reload_report` could be activated but never invoked. Defaulting closes the trap. |
